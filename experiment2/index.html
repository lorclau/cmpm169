Experiment 2: Waves - Generative methods, Fluid Dynamics, Interactivity

Description

For this experiment, I wanted to model the natural movement of calm water using generative methods. I also wanted to add some interactitiy so users can click to create ripples that distrub the water's movement.

Technical

Waves: The water is simulated with a grid whose cells are spaced out by the variable spacing and each cell has a "water height" value, which is updated at each frame to create the waves. The illusion of waves is created using a combination of a sine wave (sin(angle)) and Perlin noise (noise(i, j)). Each grid cell is drawn as an ellipse, with the size modulated by the "water height". A second smaller ellipse with reduced size is drawn on top to simulate the shading of the wave and add depth. As the frameCount increases, the waveforms evolve, creating dynamic movement in the water.

Ripples: ripples now originate from the mouse position (ripple.x, ripple.y) and affects all the grid cells based on their distance from the click position. The strength of the ripple at each grid cell decreases with the distance from the mouse click, so the closer the grid cell is to the mouse, the stronger the ripple effect; this spread of the ripples is controlled by the decayRate. The sine wave effect is still applied (sin(frameCount * 0.1 + d * 0.5)), and the wave strength is modified by the distance (waveStrength = max(0, ripple.strength - d * ripple.decayRate)). The strength of the ripple (ripple.strength) diminishes over time due to the damping factor, making the ripple fade out gradually.
